# Лабораторна робота №4: Аналітичні SQL-запити (OLAP)
**Виконав:** Шмігель Макар Олексійович, група ІО-44

Цей файл містить опис аналітичних запитів, створених для бази даних.

### Запит 1: Базова агрегація (COUNT, AVG, MIN, MAX)
**Що робить:** Обчислює загальну кількість унікальних товарів у магазині, їхню середню, мінімальну та максимальну ціну.
**Чому:** Дозволяє швидко оцінити загальний ціновий діапазон та асортимент магазину.

### Запит 2: Групування даних (GROUP BY)
**Що робить:** З'єднує таблиці `Category` та `Product` і підраховує, скільки товарів належить до кожної категорії.
**Чому:** Допомагає менеджменту зрозуміти, які категорії товарів найширше представлені на складі.

### Запит 3: Фільтрація згрупованих даних (HAVING)
**Що робить:** Рахує середню ціну товарів у кожній категорії, але виводить лише ті категорії, де середня ціна перевищує 1000 грн.
**Чому:** Дозволяє виокремити преміальні категорії (наприклад, "Електроніка"), відфільтрувавши дешеві категорії.

### Запит 4: Внутрішнє з'єднання (INNER JOIN)
**Що робить:** Об'єднує таблиці `Orders` та `Customer`, щоб замість незрозумілого `customer_id` показати реальне ім'я та прізвище замовника.
**Чому:** Необхідно для формування читабельного списку замовлень для служби підтримки чи кур'єрів.

### Запит 5: Ліве з'єднання (LEFT JOIN)
**Що робить:** Виводить абсолютно всіх клієнтів з таблиці `Customer` та приєднує до них їхні замовлення. Якщо клієнт щойно зареєструвався і ще нічого не купив, навпроти його імені стоятиме значення `NULL`.
**Чому:** Цей аналітичний запит ідеально підходить для пошуку клієнтів, які давно не робили покупок, щоб надіслати їм промокод.

### Запит 6: Комплексний запит (JOIN + GROUP BY + ORDER BY)
**Що робить:** Рахує загальну суму покупок (`SUM`) для кожного клієнта і сортує список за спаданням (`ORDER BY DESC`).
**Чому:** Використовується для визначення "VIP-клієнтів", які приносять магазину найбільше прибутку.
